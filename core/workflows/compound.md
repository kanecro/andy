# Workflow: compound

## Goal

Capture lessons and update durable knowledge after a change.

## Inputs

- Proposal
- Design
- Implementation notes
- Review summary
- Test results

## Outputs

- `docs/compound/YYYY-MM-DD-<topic>.md`
- Optional updates to policies, skills, guardrails, or artifact schemas
- Archived spec change where appropriate

## Steps

1. Read available source artifacts.
2. Extract what worked, what failed, root causes, and prevention.
3. Classify learnings by target artifact.
4. Propose durable updates; apply only approved changes.
5. Archive completed change artifacts if workflow uses OpenSpec.


## Common rules

- Follow `core/policies/escalation-policy.md`.
- Follow `core/policies/verification-policy.md` before claiming completion.
- Keep changes minimal and scoped.
- Write artifacts to the project, not to root ad-hoc notes, unless requested.
